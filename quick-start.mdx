Elysia is a TypeScript backend framework with support for multiple runtimes, optimized for Bun. The steps below walk you through getting a running server regardless of which runtime you are targeting.

<Tabs>
  <Tab title="Bun">

## Install Bun

Elysia is optimized for Bun, a fast JavaScript runtime designed as a drop-in replacement for Node.js.

<CodeGroup>

```bash macOS / Linux
curl -fsSL https://bun.sh/install | bash
```

```bash Windows
powershell -c "irm bun.sh/install.ps1 | iex"
```

</CodeGroup>

<Tabs>
  <Tab title="Auto installation">

## Create a project

The fastest way to start is with `bun create elysia`, which scaffolds a project with sensible defaults:

<Steps>
  <Step title="Scaffold the project">
    ```bash
    bun create elysia app
    ```

    This creates an `app` folder with Elysia installed and a starter `src/index.ts`.
  </Step>
  <Step title="Enter the project directory">
    ```bash
    cd app
    ```
  </Step>
  <Step title="Start the development server">
    ```bash
    bun dev
    ```

    Navigate to [http://localhost:3000](http://localhost:3000). You should see `Hello Elysia`.

    <Tip>
      The `dev` command uses Bun's `--watch` flag to reload the server automatically on file changes.
    </Tip>
  </Step>
</Steps>

  </Tab>

## Manual setup

If you prefer to set up a project from scratch, install Elysia and Bun type definitions:

<Steps>
  <Step title="Install dependencies">
    ```bash
    bun add elysia
    bun add -d @types/bun
    ```
  </Step>
  <Step title="Create the entry point">
    Create `src/index.ts` with the following content:

    ```typescript
    import { Elysia } from 'elysia'

    const app = new Elysia()
      .get('/', () => 'Hello Elysia')
      .listen(3000)

    console.log(
      `Elysia is running at ${app.server?.hostname}:${app.server?.port}`
    )
    ```
  </Step>
  <Step title="Add scripts to package.json">
    ```json
    {
      "scripts": {
        "dev": "bun --watch src/index.ts",
        "build": "bun build src/index.ts --target bun --outdir ./dist",
        "start": "NODE_ENV=production bun dist/index.js",
        "test": "bun test"
      }
    }
    ```

    - **dev** — runs with hot reload during development
    - **build** — compiles for production
    - **start** — runs the compiled output
  </Step>
  <Step title="Configure TypeScript">
    Create `tsconfig.json` or add the following to an existing one. Strict mode is required for Elysia's type inference to work correctly:

    ```json
    {
      "compilerOptions": {
        "strict": true
      }
    }
    ```
  </Step>
</Steps>

## Install Node.js

<CodeGroup>

```bash macOS (Homebrew)
brew install node
```

```bash Windows (Chocolatey)
choco install nodejs
```

```bash apt (Debian / Ubuntu)
sudo apt install nodejs
```

```bash pacman (Arch Linux)
pacman -S nodejs npm
```

</CodeGroup>

We recommend using TypeScript. Select the tab that matches your preference:

<Tabs>
  <Tab title="TypeScript">

<Steps>
  <Step title="Install dependencies">
    Elysia on Node.js requires the `@elysia/node` adapter. Install Elysia, the adapter, and TypeScript tooling:

    <CodeGroup>

    ```bash bun
    bun add elysia @elysia/node && \
    bun add -d tsx @types/node typescript
    ```

    ```bash pnpm
    pnpm add elysia @elysia/node @sinclair/typebox openapi-types && \
    pnpm add -D tsx @types/node typescript
    ```

    ```bash npm
    npm install elysia @elysia/node && \
    npm install --save-dev tsx @types/node typescript
    ```

    ```bash yarn
    yarn add elysia @elysia/node && \
    yarn add -D tsx @types/node typescript
    ```

    </CodeGroup>

    `tsx` is a TypeScript runner with hot reload and modern dev tooling built in.
  </Step>
  <Step title="Create the entry point">
    Create `src/index.ts`:

    ```typescript
    import { Elysia } from 'elysia'
    import { node } from '@elysia/node'

    const app = new Elysia({ adapter: node() })
      .get('/', () => 'Hello Elysia')
      .listen(3000, ({ hostname, port }) => {
        console.log(`Elysia is running at ${hostname}:${port}`)
      })
    ```
  </Step>
  <Step title="Add scripts to package.json">
    ```json
    {
      "scripts": {
        "dev": "tsx watch src/index.ts",
        "build": "tsc src/index.ts --outDir dist",
        "start": "NODE_ENV=production node dist/index.js"
      }
    }
    ```
  </Step>
  <Step title="Initialize TypeScript">
    ```bash
    npx tsc --init
    ```

    Then update `tsconfig.json` to enable strict mode:

    ```json
    {
      "compilerOptions": {
        "strict": true
      }
    }
    ```
  </Step>
</Steps>

  </Tab>

## WinterTC / Web Standard runtimes

Elysia is [WinterTC compliant](https://wintertc.org/), meaning it runs on any framework or runtime that supports the Web Standard `Request` / `Response` API.

<Steps>
  <Step title="Install Elysia">
    <CodeGroup>

    ```bash bun
    bun install elysia
    ```

    ```bash pnpm
    pnpm install elysia @sinclair/typebox openapi-types
    ```

    ```bash npm
    npm install elysia
    ```

    ```bash yarn
    yarn add elysia
    ```

    </CodeGroup>
  </Step>
  <Step title="Choose a runtime">
    Select a platform that supports Web Standard Request/Response:

      <Card title="Astro" icon="star" href="/integrations/astro">
        Elysia as Astro API routes.
      </Card>
      <Card title="SvelteKit" icon="code" href="/integrations/sveltekit">

      </Card>
      <Card title="Vercel" icon="triangle" href="/integrations/vercel">
        Elysia as Vercel Edge Functions.
      </Card>
    </CardGroup>
  </Step>
  <Step title="Custom runtime fallback">
    If your runtime is not listed, expose `app.fetch` as the request handler:

    ```typescript

      .get('/', () => 'Hello Elysia')

    export default app.fetch
    ```

    Any runtime that accepts a `(request: Request) => Response | Promise<Response>` function is compatible.
  </Step>
